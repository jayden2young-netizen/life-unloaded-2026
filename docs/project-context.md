# 项目开发约束

## 数据与版本

- 玩家内容只改 `content/zh-CN/` 与 `tools/generate-v5-data.mjs`，`data.json` 仅由生成器写出。
- 同一发布中的 `index.html`、`game.js`、生成器、`data.json`、README 与测试版本断言必须一致。
- 生成器连续运行两次，第二次不得产生差异。

## 事件簇

- 每阶段依次显示情况卡、选择卡和结果卡；三张卡分别点击，但阶段完成前不增加年龄。
- 每阶段最多一次关键选择；短期簇最长五年，达到截止年龄必须播放明确结尾。
- 同时最多两个事件簇，同 lane 不并行；人物和单位在启动时绑定。
- 绑定人物、单位或资格失效时，用具体物件和事实收尾，不使用泛化的“状态改变”。
- 职业、婚姻、育儿、慢性病和恢复期是长期状态，不组成几十年活动链。

## 内容与研究

- 制度、法律和医学规则以A级可靠资料校正；媒体用于常见矛盾，论坛和社交媒体只提供生活细节灵感。
- 不复制可识别私人经历，不把离奇个案写成普遍规律，不把当前政策永久写死在远期年代。
- 普通饮酒、游戏、购物、用药、远程工作或关系摩擦不能自动判为危机。

## 验证边界

- 先验证验收器与当前数据结构一致，再判断实现是否失败。
- 每个版本做静态契约、定向成功／失败／退出／失效结尾与一次真实浏览器核心路径。
- 涉及卡片或抽屉时检查 360×773、360×640、320×568；控制台错误为0。
- 不运行批量人生模拟；完成标准满足后停止。
